API开发系列-Token机制

2016-03-22
PHP
API

Token（计算机术语）在计算机身份认证是令牌的意思，在此法分析中是标记的意思，我们通过Token机制能够有效的保护接口API不被恶意‘骚扰’。

我们可以想象一下，如果一个API在没有任何防护机制的情况下是多么恐怖的一件事情。通过抓包工具开发者可以轻松的抓到APP的接口,在没有保护的情况下一些‘坏家伙’就会开始跃跃欲试了，拥有一定安全意识的人都应该知道会发生什么。

那么现在我们开始讲述如何实现一个简单的PHP实现API接口中的Token机制。

移动端和服务端的通信需要2把钥匙：

  1. 接口token（api_token）
  2. 客户端token（user_token）

接口token（api_token）

它的职责是保持接口访问的隐蔽性和有效性，保证接口的唯一性，实现的方法有很多种，我们可以按照服务端和客户端都拥有的共同属性生成一个随机串，客户端生成这个穿，服务器也按同样的算法生成一个串，用来验证客户端的串。

我使用的风格为：

api_token = md5(‘action’,’date’,’key’)

详细解析：

action：方法

date：当前时间

key：密匙

其中key为自由的加密key，手机端需要在服务端注册一个‘接口使用者’账户后，系统会自动凤佩一个账号以及密码，数据表设计参考如下：

字段名 字段类型 注释
client_id varchar(20) 客户端ID
client_secret varchar(20) 客户端（加密）key
